ReDialog, 比window.alert()更好用！

在WEB开发中，无论服务器端使用什么技术，JSP, ASP, PHP, CGI, ASPX...都势必要将一些信息展现给客户，最普通的应用莫过于用户进行一个操作之后，服务器响应给客户一个消息。而在这之前，Web开发工程师们采用的最常用的方式是使用Javascript：window.alert(...)产生一个消息框，提示用户。消息框的样式是操作系统定义好的，无法格式化，也无法美观...

1、ReDialog简介

在Web开发中，经常需要向客户端显示一些信息。在以前，window.alert()是经常被使用的。但是，Web开发工程师大多对window.alert()深恶痛绝，因为不但难以美化其外观，也不能对里面显示的文字进行颜色、大小等设置，这样给用户的是一个灰色而很不友好的小窗口，伴随着“当”的一声，出现在用户的面前，实在很不友好。对此Web工程师也没有好的办法----毕竟是操作系统的东西，没办法啊。

ReDialog的出现就是为了解决这个问题。ReDialog是采用客户端JavaScript技术，结合弹出窗口，实现了更为友好的界面（如图）。更重要的是，他不依赖于任何一种服务器脚本，如ActiveServerPage, Java Server Page, PHP, Perl, CGI等，可以很轻松的将现有的Web应用程序升级到ReDialog中。易于配置和维护，更大的自由度，使得ReDialog成为Web开发客户端消息响应显示的首选。

标准与客户交户的函数有三个：alert, confirm 和prompt。这些函数在ReDialog中都可以找到对应，并且它的使用方式、感觉、返回值等与它们基本相同。在一般情况下，开发人员不需要将源程序进行太大的改动就可以将Web程序升级为更为美观的ReDialog了。

ReDialog是一个开放源代码的项目，任何个人、商业组织都可以不受限制的使用、分发、修改其中的源程序为自己所用。作者也将不断的更新这个项目使之更加友好、方便。

2、ReDialog的特点

  • 友好的界面，在一般情况下，ReDialog能够更友好的与客户交户
  • 良好的兼容性。回车键(ENTER)、跳出键(ESC)的处理均与标准的Windows操作保持一致。
  • 纯JavaScript设计，最大程度的与服务器脚本无关。
  • 易于配置，你只需要更改一个参数即可让ReDialog正确运行。
  • 个性化设置自由。一般情况下，只需要简单的更改两三个参数就可以设置出符合你自己的Web应用程序的外观。
  • 免费。任何个人、商业组织都可以不受限制的使用、分发、修改其中的源程序为自己所用。

3、ReDialog的配置

  • 下载最新的ReDialog程序 http://www.jzchen.net/redialog/
  • 下载的程序为redialog.zip压缩包，解压缩到你想解压缩的地方
  • 假定你将解压缩后的redialog文件夹位于到"web程序根目录/redialog"
  • 打开redialog/redialog.js， 将“rdHome”变量更改为redialog的实际路径。注意：在路径的最后必须有一个"/"， 不然出错。
  • （可选）打开redialog/rd-config.js， 设置相关变量，使之适应自己的Web程序
  • 在你想加入ReDialog的网页中的<head></head>之间加入script应用：
    <script language="javascript" src="/redialog/redialog.js"></script>
  • 最后，按照下面介绍的API说明，尽情使用ReDialog吧！
